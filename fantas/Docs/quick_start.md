# 快速使用 fantas

## fantas 简介

fantas 是一个基于 pygame 的 2D 图形框架，准确的说，是 pygame 的社区版 pygame-ce。

> pygame 原本是由一群人维护的开源项目，但是官方版的网站、仓库、社交媒体账号等权限高度集中在一个人的手里，而这个人在 pygame 的开发方向上与其他大部分核心开发者有着较大的分歧，并且他还一度关停网站服务器以达到某些政治目的，这使得官方版的开发收到了极大的阻力。pygame 官方的最后一次 release 停留在了 2024年6月25日（版本2.6.0）。
不过，开源的魅力在于你不需要受制于人，于是众多社区开发者 fork 了一个仓库版本，从官方版独立了出来，形成了社区版本 pygame-ce。社区版保留了 pygame 原本的所有接口，原本使用官方版的程序可以无痛切换到社区版（甚至连包的名字都不用改，仍然是`import pygame`）；但是社区版保持着活跃的更新与修复，新增了一系列实用的功能，以及性能优化。

如果你使用过 pygame，你应该会很熟悉 fantas 的很多概念；如果你没有，别担心，fantas 上手更简单，并且完全隐藏了 pygame，你不需要自己使用 pygame 的接口；如果你连 pygame 是什么都不知道，你可以去看一下他们的 [官方文档](https://pyga.me/docs/index.html)（这个链接当然是 pygame-ce 的文档）。

为什么要在 pygame 的基础上开发 fantas 呢？因为 pygame 作为一个图形库，提供的核心功能其实相当底层——你需要自己绘制窗口里面的所有内容（每一个像素点），然后自己拿到事件作处理，就连主循环也要自己写。简单点说，就是不存在“一句话创建一个按钮”这种写法，你需要自己画一个矩形，再画一行文字，然后拿到鼠标点击事件，判断坐标是否落在你画的矩形内部，如果是，那么执行按钮的功能。这一套操作相当繁琐，不过，如此底层的配置却提供了相当高的自由度，于是在此之上，我开发了一套简单易用的图形框架。

虽然 pygame 适用于开发游戏的，但是 fantas 并没有按照游戏开发的思路去搭建框架，而是以通用图形化交互程序为目标设计的，因此不仅仅用于游戏开发，还能胜任一般化的图形程序。

## 直接上手

目前还不能通过 pip 下载（正在计划中），你需要将 fantas 文件夹放在代码根目录下（或者任何能够在 import 时被找到的目录下）。

前面已经说了，使用 fantas 十分简单：

``` python
import fantas

window_config = fantas.WindowConfig()
window = fantas.Window(window_config)

window.mainloop()
```

运行上面这段代码你就可以得到一个窗口了，什么？你想要自定义窗口的样式，只要在初始化 `WindowConfig` 时提供关键字参数即可：

``` python
window_config = fantas.WindowConfig(
    title = "Fantas3 Demo",
    window_size = (1920, 1080),
)
```

更多参数及对应功能可以参考详细的文档说明。总之，你应该能够感受到 fantas 的优雅之处。虽然进行了高度的封装，但是仍然保留了 pygame 程序应有的自由度，你可以用 fantas 实现所有 pygame 能够实现的效果，而且更简单。

上面只是创建了一个窗口，下面给出一个完整的示例，看看 fantas 的日常吧：
